# Contributing to reachy-mini-agent

Thanks for thinking about contributing! This project is small and pragmatic — PRs that fix a specific problem or land an incremental improvement are welcome. Sprawling refactors without an issue attached will likely be asked to split.

## Quick checklist before opening a PR

- [ ] The change has a clear, narrow scope (one bug / one feature).
- [ ] `python -m py_compile src/*.py` succeeds.
- [ ] `ruff check .` is clean (or you explain the remaining warnings).
- [ ] You've tested it end-to-end with a real or simulated Reachy Mini *or* clearly marked the PR "untested against hardware."
- [ ] Any new env var is documented in `.env.example` and `README.md`.
- [ ] Commit messages are imperative, descriptive, and contain no AI / bot signatures ("Co-Authored-By: Claude", "Generated by …", etc.).

## Development setup

```bash
git clone https://github.com/thc1006/reachy-mini-agent.git
cd reachy-mini-agent
uv venv --python 3.12
uv pip install -e ".[servers,kokoro]"
uv pip install ruff pytest    # dev tools
```

See [docs/SETUP.md](docs/SETUP.md) for the full runtime prerequisites (gst-plugins-rs, Ollama, model downloads).

## Running tests

```bash
pytest -q
```

Tests are intentionally lightweight smoke tests (import + pure-function checks). Hardware-in-the-loop tests need the robot connected and are not part of CI.

## Style

- Black-compatible formatting, 99-column soft limit.
- Chinese comments are allowed and common in this codebase (it's a bilingual project); English is preferred for public API docstrings.
- No extraneous comments describing *what* the code does — only *why*, when non-obvious.
